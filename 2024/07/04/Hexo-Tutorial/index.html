Hexo 教程

暂时只是搭建一个简单的Hexo Blog 还有一些功能还没研究 后续研究会更新文档

也可以参考Hexo官方文档

安装nodejs

  1. 去Nodejs官方下载 nodejs
  2. 按照安装步骤一步步安装好 安装好了之后使用node -v检测是否完成(可能需要检查环境变量)

安装Hexo

  1. 下载hexo客户端 npm install hexo-cli -g
  2. 本地创建blog文件夹 mkdir my_blog & cd my_blog & hexo init 或者直接hexo init my_blog
  3. 下载依赖 npm install

初始化之后会出现如下的文件

.
├── _config.yml
├── package.json
├── scaffolds
├── source
|   ├── _drafts
|   └── _posts
└── themes

_config.yml: 配置文件 hexo的配置信息 需要修改部分

package.json:应用程序的信息

scaffolds:模版文件夹。新建文章时，Hexo 会根据 scaffold 来创建文件

source: 存放用户资源的地方也就是存放写的文章

thmems: hexo主题 可以下载一些喜欢的样式

配置GitHub

如果想要部署在github上(推荐) 需要在自己github上创建仓库

仓库名必须是 username.github.io

创建完之后应该是一个的仓库(不知道怎么创建的可以去了解一下github创建仓库的流程 以及配置自己的github比如username/email…)

配置github.io domain username.github.io.git

基础操作

hexo n "Article Name" 会在source/_post下新建一篇文章 markdown格式的

hexo clean 清楚清除缓存文件 (db.json) 和已生成的静态文件 (public)。 在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令。

hexo g 生成静态文件

hexo d 部署到github上

hexo s本地启动服务

一般生成好文件之后需要先保存文章 在生成静态文件 然后可以本地启服务先看 最后在部署到github

设置可以写个sh脚本一键部署

#!/bin/sh

# Clean the Hexo generated files
hexo clean

# Generate the static files
hexo g

# Deploy the generated files
hexo d
