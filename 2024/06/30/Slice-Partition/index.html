数组分和——动态规划

416. 分割等和子集

给你一个 只包含正整数 的 非空 数组 nums 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

示例 1：

输入：nums = [1,5,11,5]
输出：true
解释：数组可以分割成 [1, 5, 5] 和 [11] 。

示例 2：

输入：nums = [1,2,3,5]
输出：false
解释：数组不能分割成两个元素和相等的子集。

提示：

  • 1 <= nums.length <= 200
  • 1 <= nums[i] <= 100

分析

动态规划

  1. 定义dp[i][j] 表示 从[0, i]中是否存在和为j的子序列
  2. 状态转移
    1. 第 i+1 个数不选时 则 dp[i+1][j] = dp[i][j]
    2. 选i+1个数时 则 dp[i+1][j] = dp[i][j-nums[i]]
  3. 初始状态 dp[0][0] = true
  4. 最终结果 dp[n][sum/2]
func canPartition(nums []int) bool {
    n := len(nums)
    sum := 0
    for i := range n {
        sum += nums[i]
    }
    if sum%2 != 0 {  // 不能整除说明无法二等分
        return false
    }
    setSum := sum/ 2  // 每个子序列的和
    // dp[i][j] 表示前 [0, i]中是否有和为j的子序列存在
    dp := make([][]bool, n+1)
    dp[0] = make([]bool, setSum+1)
    dp[0][0] = true
    // 状态转移 dp[i][j] = dp[i-1][j] || dp[i-1][j-nums[i]]  不选i这个元素 以及选i这个元素
    for i := range n {
        dp[i+1] = make([]bool, setSum+1)
        for j := 0; j <= setSum; j++ {
            dp[i+1][j] = dp[i][j] || (j >= nums[i] && dp[i][j-nums[i]])
        }
    }

    return dp[len(nums)][setSum]
}

698. 划分为k个相等的子集

给定一个整数数组 nums 和一个正整数 k，找出是否有可能把这个数组分成 k 个非空子集，其总和都相等。

示例 1：

输入： nums = [4, 3, 2, 3, 5, 2, 1], k = 4
输出： True
说明： 有可能将其分成 4 个子集（5），（1,4），（2,3），（2,3）等于总和。

示例 2:

输入: nums = [1,2,3,4], k = 3
输出: false

提示：

  • 1 <= k <= len(nums) <= 16
  • 0 < nums[i] < 10000
  • 每个元素的频率在 [1,4] 范围内
func canPartitionKSubsets(nums []int, k int) bool {
    sum := 0
    for _, v:= range nums{
        sum+=v
    }
    if sum % k != 0{
        return false
    }
    perSum := sum/k
    sort.Ints(nums)
    if nums[len(nums) - 1] > perSum{  // 如果最大的比perSum大 那么这个数的组合不可能=perSum
        return false
    }
    visited := make([]bool, len(nums))
    var dfs func(target, k, start int, visited []bool) bool
    dfs = func(target, k, start int, visited []bool) bool{  // 要找k-1次 每次找一个和为perSum的子序列
        if target == 0{  // 说明找到一组满足条件的子序列 继续找下一组
            return dfs(perSum, k-1, 0, visited)
        }
        if k == 0{  // 所以组合都找到了
            return true
        }
        for i:= start; i<len(nums); i++{
            if visited[i]{
                continue
            }
            visited[i] = true
            if dfs(target - nums[i], k, i+1, visited){
                return true
            }
            visited[i] = false  // 回溯
        }
        return false
    }
    return dfs(perSum, k-1, 0, visited)
}

时间复杂度分析

详细分析

算法步骤

  1. 计算数组总和：时间复杂度为 O(n))，其中 n 是数组 nums 的长度。
  2. 检查总和是否能被 k 整除：时间复杂度为 O(1)。
  3. **计算每个子集的目标和 perSum**：时间复杂度为 O(1)。
  4. 对数组进行排序：时间复杂度为 O(nlog⁡n)。
  5. 初始化 visited 数组：时间复杂度为 O(n)。

深度优先搜索 (DFS) 部分

DFS 的核心部分是一个递归函数 dfs，其中包含一个 for 循环和递归调用。

  • 目标和 target 和子集数 k 的减少方式：
    • 每次成功找到一个和为 perSum 的子集后，k 减 1，继续寻找下一个子集。
    • 当 target 为 0 时，重新开始寻找 perSum 的子集，但 k 减 1。
  • 遍历和递归：
    • 每个递归调用都会遍历剩余的元素，这些元素数量在最坏情况下为 O(n)。
    • 在每次递归调用中，最多会进行 O(n) 次递归调用。
  • 回溯：
    • 在找到一个满足条件的子集后，会进行回溯，尝试其他可能性。

更详细的时间复杂度分析

考虑每次递归调用中，每个元素都可以被选择或者不被选择，形成一种组合的情况：

  • 在最坏情况下，所有组合都会被探索，即 2^n 种组合。
  • 但是由于剪枝操作（例如如果当前和已经超过 target，则不会继续递归），实际的复杂度会低于 2^n。

对于每一个组合，我们需要检查是否可以构成 k 个子集，因此最坏情况下，时间复杂度接近于： O(k * 2^n)

综合时间复杂度

由于 O(k * 2^n) 是更大的项，因此我们可以忽略排序的时间复杂度 O(nlog n)。综合上述分析，该算法的时间复杂度为:O(k * 2^n)

其中，n 是数组 nums 的长度，k 是子集的数量。
